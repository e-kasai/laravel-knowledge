### 役割
文字列の大小比較

### ex
```php
// 時刻でソート（早い順）
        usort($validBreaks, function ($a, $b) {
            return strcmp($a['break_in'], $b['break_in']);
        });
```

### 解説
文字列比較の仕組みが「数字として」ではなく「文字コード（バイトの順序）」で行われる

たとえば PHP の `strcmp("09:00", "10:00")` はこう動きます。

1. 先頭の文字 '0' と '1' を比べる
2. `0` の文字コードは `48`、 `1` は `49`
3. `48 < 49` なので "09:00" のほうが「小さい」と判断される
4. よって "09:00" が「先の時間」として扱われる

つまり
文字列の先頭から順に1文字ずつ文字コードを比較し、最初に違いが出たところで順序を決める

### 注意点
FRでゼロ埋め（date_format:H:i）にフォーマットされていないと順序が狂う

ex:
ゼロ埋めがない場合

例： "9:00", "10:00"
これを ゼロ埋めなしのまま`strcmp()` で比べると：

"9:00" vs "10:00"	    "9:00" の方が大きい（誤り）  	'9' (57) > '1' (49)
