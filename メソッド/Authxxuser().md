
Auth::user()


### **1. 正体**

`Auth` は **Facade（ファサード）** という仕組みで、  
実際には `Illuminate\Support\Facades\Auth` クラスを通じて  
`AuthManager` や `Guard` という認証クラスのメソッドを呼んでいます。

だから

`Auth::user()`

は **Authクラスの `user()` メソッドを呼び出している** 形です。

---

### 2. 返り値

ログイン中なら **`User` モデルのインスタンス** が返ります。  (ログイン中のユーザー１人分の情報)
未ログインの場合は `null`。

※全部のユーザーのプロフィール一覧がほしいなど複数の場合は[[with()]]を使う
```php
$user = Auth::user(); 
	if ($user) {
	     echo $user->name; 
	     }
 ```
---


### 3.インスタンス経由との違い

同じ処理をこう書くこともできます。

`auth()->user();`

`auth()` は `Auth` ファサードの **インスタンスを返す関数** で、  
そのインスタンスに `->user()` を呼び出しているだけ。  
動きは同じです。

---

### **4. よくある呼び出しの比較**

|書き方|意味|
|---|---|
|`Auth::user()`|ファサード経由で `user()` メソッド呼び出し|
|`auth()->user()`|ヘルパー関数経由で `user()` メソッド呼び出し|
