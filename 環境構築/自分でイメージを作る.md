

## コンテナ作成までのフロー

## 1. Dockerfile を作成

今回は、`~/practice`ディレクトリ以下に`dockerfile-practice`ディレクトリを作成し、環境構築をしてみる

```
touch Dockerfile
```

同じ階層状に `html` ディレクトリを作成し、その中に `test.html` というファイルを作成

```
mkdir html
touch html/test.html  
```


## 2.Dockerfile の記述

例
```html
FROM nginx                            //イメージを指定
COPY ./html /usr/share/nginx/html     //html ディレクトリごとコピーして、 コンテナ内の/usr/share/nginx/html という階層に対して複製
```


## 3. イメージのビルド（Dockerfileからイメージを作成する）

注：Dockerfile が置いてあるディレクトリから実行

```bash
// docker build -t [生成するイメージ名]:[タグ名] [Dockerfileの場所]

docker build -t nginx-image .     //このイメージに nginx-image という名前を付けて作る
```
-t = --tag (イメージに名前）を付ける オプション


## 4. 作成したイメージからコンテナを作成

```bash
docker run -d --name nginx-container -p 80:80 nginx-image
```


`--name` で作成するコンテナの名前を付ける
`-d` はコンテナの実行ログをターミナルに出力させないオプション
`-p` がポート番号を割り当てるオプション　（左側がホスト、右側がゲスト側（docker）のOSのポート番号）
