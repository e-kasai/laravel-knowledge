### 説明
- 認証なし（純粋なLaravel＋Docker＋基本設定）
- 課題や検証で「素の状態」をすぐ使える

---
### step1　教材の環境をclone
- 教材から環境をclone

```
cd project_files
git clone git@github.com:Estra-Coachtech/laravel-docker-template.git

mv laravel-docker-template laravel-template-base
```

---

### step2 clone元との紐づけを切る
```
cd laravel-template-base
git remote -v #現在のpush先を確認
git remote remove origin #紐づけを削除
git remote -v #なにも表示されなければOK

```

### step3  .gitディレクトリ削除

- エクスプローラから`.gitディレクトリ`を削除
※clone元の.gitがついてくるので別ディレクトリとして扱うためには一度.gitを削除してgit initが必要

---

### step4 新しいremoteリポジトリを作成

- GitHubで `laravel-template-base` という空リポジトリを作成

---

### step5 必要なファイルを追加

- `README.md`
- `docker/mysql/data/.gitkeep`   
- ~~`docsディレクトリ`　project直下に作成し`er.png`というファイル名でER図を置く為~~(からディレクトリ無視されるので後でいい)
- `.gitignore(project直下)`   Dockerのログやvendor、node_modules、環境ファイルなどを除外
- `.gitignore(src下)`   念のため必要なファイルが含まれているか確認

#### 補足

- `src/storage/...` 系や `src/bootstrap/cache` 系はLaravelが最初から `.gitignore` を配置してくれる

---
### step5 作成したリポジトリにpush

```
git init
git branch -M main
git remote add origin git@github.com:<ユーザー名>/laravel-template-base.git
git add .
git commit -m "Initial commit for laravel-template-base"
git push -u origin main
```
