
---

## 1. Laravelのコード

実体：PHP・Blade・CSSなどのコード全部
保存場所：PCの プロジェクトフォルダ（Git管理）
消すとどうなる？：単にコードがなくなる。Docker環境は残るけど中身がない。

🟢 整理時ポイント：
→ プロジェクトがもう不要なら削除してOK
→ まだ触るかもなら残しておく（安全・軽い）


##  2. PHP / Nginx コンテナ（実行環境）

実体：Laravelを動かす「仮想環境」
保存場所：Docker内部
削除しても docker compose up すれば自動で再生成される。

🟢 整理時ポイント：
→ 停止中なら docker container prune (停止しているコンテナを全部まとめて削除する) で消してOK
→ コードもボリュームも残るから、再起動( `docker compose up -d`)すれば元どおり
→ 少しは軽くなる、“劇的に”軽くはならない

---

## 3. MySQLコンテナ（DBを動かす仮想環境）

実体：MySQLが動いている仮想環境
保存場所：Docker内部
中身（DBデータ）はボリュームに保存される

🟡 整理時ポイント：
→ コンテナ削除はOK（docker container prune）
→ ただし、ボリュームを削除するとDBの中身（実際のDBのデータ）も消える

---

## 4. ボリューム（mysql_data）

実体：MySQLの中のデータファイル（テーブル・レコードなど）
保存場所：ホスト上（/var/lib/docker/volumes/...）
コンテナから独立して存在

🔴 整理時ポイント：
→ docker volume prune は要注意！
→ データベースの中身がゼロになって、再マイグレーション・再シーディングが必要
