### 結論

Fortify の ログアウト後の遷移先は `config/fortify.php` で設定できる。（LogoutResponse を自作不要）

### デフォルトの挙動

状況	　　　　　　　　　　　　　遷移先　　　　　　　　　	備考
設定なし　　　　　　　　　　	/	　　　　　　　　　　　　　　Fortify標準のデフォルト（トップへ）
設定あり（推奨）　　　　	/login	　　　　　　　　　一般的なログアウト後の挙動


### 設定方法

ファイル: `config/fortify.php`
```php
'redirects' => [
    'logout' => '/login',
],
```


この設定で、/logout 実行後に /login へ自動リダイレクトされる。
Fortify が内部で `redirect(Fortify::redirects('logout', '/'))` を呼び出すため、この設定が最終的に反映される。

---

### Fortify挙動解説

Fortify のログアウト処理は `vendor/laravel/fortify/src/Http/Controllers/AuthenticatedSessionController.php @destroy` に定義されている。

その末尾で以下のように呼ばれる：
```php
return app(LogoutResponse::class);
```

`vendor/laravel/fortify/src/Contracts/LogoutResponse.php` の中身は次のようになっている：
```php
return $request->wantsJson()
    ? new JsonResponse('', 204)
    : redirect(Fortify::redirects('logout', '/'));
```
`LogoutResponseクラス`の完全修飾名は`Laravel\Fortify\Contracts\LogoutResponse`


- つまり：
JSONリクエスト → `204 No Content`
通常リクエスト → `config/fortify.php` の logout 設定先にリダイレクト
