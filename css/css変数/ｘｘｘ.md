/*---------------------------------
留意ポイント

1 レスポンシブ対応度を上げるため可変単位のみ使用する
2 ヘッダー内の要素配置はposition relativeではなくgridで安定性を上げる
3 rem,vh,vw,%の使い分けを意図的に行う
- rem     padding, margin, font-size
- vh,vw   幅や高さ（親要素の影響を受けない = 画面全体が親とみなせる場合）
- %       幅や高さ(親要素のサイズによって影響される場合)
---------------------------------*/

以下気になった点です。  
■ CSS変数の導入  
保守性を考慮して、変数を定義してあげるとよりよいと思います。  
▽ 例：  

:root {
    /* カラーパレット */
    --color-primary: #d9c6b5;
    --color-secondary: brown;
    --color-background: #f1ece7;
    --color-surface: white;
    --color-border: #e0dfde;
    --color-border-light: #c5b9b0;
    --color-focus: #6c63ff;
}

■ その他気になった点  

- `clamp()`の値をCSS変数化して統一性を向上
- `.register-form__group`に`:last-child`を使用してマージンの調整
- エラーメッセージ用のスタイルクラスの準備
- 視認性を上げるといった観点でボタン系の要素にホバー時の動きを追加するとよりよいかと思います。
