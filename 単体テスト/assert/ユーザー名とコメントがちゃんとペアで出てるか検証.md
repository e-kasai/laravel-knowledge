#### テストcode
```php
$html = $response->getContent(); 
$this->assertMatchesRegularExpression('/Alice.*ナイスな商品ですね/s', $html); 
$this->assertMatchesRegularExpression('/Bob.*購入を検討しています/s', $html);
```

---

#### コード解説

1. $html = $response->getContent();

ここでレスポンスのHTML全体を文字列として取り出してるの。
つまりテストで実際にブラウザに返されるHTMLが $html に入る。


2. $this->assertMatchesRegularExpression(...)

PHPUnit のメソッドで、「この正規表現にマッチする文字列が $html にあるか」をチェックするものよ。
Laravelの assertSeeText より柔軟で、テキストの順序や近さを検証できる。



3. 正規表現の中身：/Alice.*ナイスな商品ですね/s

/ ... / : 正規表現の区切り

Alice : 「Alice」という文字列を探す

.* : 「任意の文字が0文字以上」を意味する。つまり「Aliceのあとに何かあってもいい」

ナイスな商品ですね : この文字列が続くことを要求

/s : 修飾子。「ドット.が改行も含む」ようになる。これで複数行にまたがってもOKになる

つまりこれは：
「HTMLの中で、Aliceの後に『ナイスな商品ですね』が続いていることを保証」
