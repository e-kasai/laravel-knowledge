1. MySQLコンテナからMySQLに、rootユーザでログインして、demo_testというデータベースを作成
	実行場所：MySQLコンテナ内

```bash
docker exec -it mysql bash   // ← mysqlコンテナ名が "mysql" の場合    

// コンテナの中で MySQL にログイン
$ mysql -u root -p //docker-compose.ymlで設定したMYSQL_ROOT_PASSWORD, userはroot(管理者)

//注！このDB 名は.env.testingのDB_DATABASE= / phpunit.xmlの<server name="DB_DATABASE" value=と一致する必要がある
CREATE DATABASE demo_test;  
SHOW DATABASES;    // ここで demo_test が表示されればOK
```

#### point
- なぜrootでやるか？
データベース作成は権限が必要だから。app用ユーザーはDB操作が制限されてることが多い

#### テスト用DBは１つでOK?
- 単体/機能テストを順番に実行する　：demo_test 1個＋毎回リセット（RefreshDatabase）で十分。
- 並列実行（--parallel）する　　　　：demo_test_1, demo_test_2 … のようにプロセス数分用意（テストが数百件など多い時）
