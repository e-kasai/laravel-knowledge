#### 手順

- テスト用テーブルのマイグレーション実行
```bash
//phpコンテナ内
php artisan migrate --env=testing
```

#### マイグレーションfileは？？

- Laravelのテストは 本番/開発用と同じマイグレーションファイルを使って DB を構築する仕組み = 専用のmigration file不要
- use RefreshDatabase; を使っていると、テスト前に 指定されたDB（demo_test）を migrate:fresh でリセット＆再構築

→ 既存の database/migrations のファイルが全部流れる
