
---

#### アクセサとは

* 役割：DBに入ってる値を「人間に見せる用」に変換する仕組み
* 書き場所：モデルクラス（app/Models/...）の中、リレーションの下
* bladeでの呼び出し方：$model->property_name で使える
* DB保存前に加工するのがミューテタ、DB保存値をあとから見やすく加工するのがアクセサ

---

####  命名ルール(重要)

形式：get{Name}Attribute()
参照名：snake_case に変換される。

`getConditionLabelAttribute` → `$item->condition_label`
`getFullNameAttribute` → `$user->full_name`

例）get`ConditionLabel`Attribute(モデル側) → $item->`condition_label`(blade側)

---

#### 例：状態コードを文字に変換

```php
// app/Models/Item.php
class Item extends Model
{
    public function getConditionLabelAttribute(): string
    {
        return match ($this->condition) {
            1 => '新品・未使用',
            2 => '目立った傷や汚れなし',
            3 => 'やや傷や汚れあり',
            4 => '状態が悪い',
            default => '不明',
        };
    }
}
```

```html
<span>状態</span>
<span>{{ $item->condition_label }}</span>
```

---
#### まとめ
* DBには数字（1,2,3…）を保存 → 表示時は文字でわかりやすくできる
* Bladeにロジックを書かず、モデルにまとめるのがスマート
* 似た概念で ミューテータ（Mutator） もある
	アクセサ = 「取り出す時に整形」
	ミューテータ = 「保存する時に整形」
