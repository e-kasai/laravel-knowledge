#### stripeアカウント作成

Stripe公式サイトにアクセス
https://dashboard.stripe.com/register

メールアドレス(個人のgmailなどでOK)　と パスワードでアカウント作成
ログイン後はテスト用の「開発モード（テスト環境）」がすぐに使える。実際のお金は動かないから安心。

事業者情報を入力（本番用のみ）
* 本番（実際にお金を受け取る）に移行するには、銀行口座や事業者情報の登録が必要。
* 学習・開発中ならテストモードだけで十分。

つまり、開発中のフリマアプリに Stripe を組み込むだけなら 無料アカウント＋テストモード でOK。


---

* メールアドレス　　個人のgmailなどでOK
* 氏名　　　　　　　開発のみ：ニックネームなどでもＯＫ
　　　　　　　　　　　本番：本名（個人事業主なら開業届の名前、法人なら法人名）
* ビジネス名　　　　開発のみ：適当で大丈夫　
　　　　　　　　　　　　本番：顧客の決済明細に載るから、本名や屋号、法人名など
* WebサイトURL　　開発のみ：なくても可
　　　　　　　　　　　　　本番：実際のURL
* 使用する機能　　　用途に応じて（今回のフリマアプリでは継続支払い以外）


アカウントの設定が終わったらサンドボックス（stripeのテスト環境画面）に進んでOK

#### APIキーの確認

* 左メニューの一番下にある 「開発者」 をクリックして、その中の 「APIキー」 を開く
* ここで「公開可能キー（pk_test〜）」と「シークレットキー（sk_test〜）」が見える

** これを Laravel 側の .env に書いて使う **

---

* 公開可能キー　（フロント側（JavaScriptなど）で使ってOKな公開鍵）
```
pk_testから始まるキー
```
* シークレットキー　（サーバー側（Laravelのコントローラーなど）で使う秘密鍵、github公開絶対NG）
```
sk_test_から始まるキー
```
