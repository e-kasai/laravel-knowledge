

## 今日やったこと

### 1. Dockerコンテナの基本操作
- `docker run` を使って Apache（httpd）コンテナを起動
- `docker stop` / `docker rm` で停止・削除の流れを確認
- `docker ps` / `docker ps -a` で状態確認

---

### 2. バインドマウントの理解と実践
- ホスト（Windows / WSL）とコンテナのパス対応を確認
  - `C:\Users\ek-12\Documents` → `/mnt/c/Users/ek-12/Documents`
- `-v ホストパス:コンテナパス` を使ったバインドマウントを実施
- Apache の公開ディレクトリ `/usr/local/apache2/htdocs` にマウント
- ホスト側で `index.html` を編集 → ブラウザ表示が即反映されることを確認
- 「編集して即反映＝マウント成功」という判定基準を体感

---

### 3. 名前付きボリュームの理解と実践
- `docker volume create apa000vol1` で名前付きボリューム作成
- `-v ボリューム名:コンテナパス` を使ってマウント
- 名前付きボリュームはホストから直接見えないことを確認
- コンテナ内でファイルを作成し、ブラウザ表示で動作確認

---

### 4. データの永続化の理解
- データの永続化＝「コンテナを消してもデータが残る状態」
- 永続化は目的、マウントはその手段であることを整理
- バインドマウントと名前付きボリュームの違いを理解

---

### 5. 後片付け
- コンテナ削除後もボリュームは残ることを確認
- `docker rmi httpd` でイメージ削除
- `docker volume rm apa000vol1` でボリューム削除
- Docker環境をクリーンな状態に戻した

---

## 今日の理解ポイントまとめ
- マウントとは「コンテナ内と外の保存場所をつなぐこと」
- データの永続化は“状態”、マウントは“方法”
- `docker run` は IMAGE を最後に必ず指定する
```
docker run --name apa000ex21 -d \
  -p 8091:80 \
  -v apa000vol1:/usr/local/apache2/htdocs \
  httpd  //イメージ
```
  
- 相対パスは実行ディレクトリ依存、絶対パスは依存しない
- 困ったら「コンテナの中から `ls` して確認」
