
---

## ■ 基本の流れ

作業はつねに1方向 に流れる。
```
feature → develop → main
```

---

## ■ 各ブランチの役割

### ● feature（作業用）
- 自分が作る機能ごとに１つ作る。
- 作業が終わったらPRを出して消える使い捨て
- 例：`feature/add-login`, `feature/refactor-cart`


### ● develop（チーム集合場所）
- みんなの作業置き場（共有ブランチ）
- PRの行き先は基本ここ
- 「実務で動く最新バージョン」はここ


### ● main（本番）
- 公開されている安定版
- 事故防止のため自分で直接触らない
- 運用チームがリリース時に develop を取り込むことが多い

---

## ■ まとめ

- 作業は **feature**  
- 最新は **develop**  
- 本番は **main**  

そして流れは **feature → develop → main**
