# ルール

1. ブランチ切る前に main を最新にする 
→　最新の状態を土台にブランチを切らないとコンフリクト地獄になる
→　ほかの人の作業（最新のmain）と違くなる

2. 作業前にbranchを切る = 未コミットの状態でbranchを切らない
→ switch -cでも未コミットは持ち越すが、切り替え先の状態と衝突するとエラーが出るため。[^1]

誤ってbranchきらず作業してしまった→参照：[[作業前にbranchを切り忘れた(mainで作業してしまった)]]

## ■ 手順（プロジェクトのルートで実行）

Excel参照


[^1]: 例
	
	今のブランチ（feature/chat）で：
	app.blade.php の 10行目をこう変えた：
	
	<h1>Hello</h1>   → <h1>こんにちは</h1>
	
	切り替え先の main ブランチでは：
	<h1>Hello</h1> → <h1>Welcome</h1>
	
	同じ行に違う変更がある状態。
	
	このときに git switch main すると…
	
	error: Your local changes to the following files would be overwritten:
	app.blade.php
	
	というエラーが出る。
	
	Git は
	「そのまま切り替えるとどっちの変更も失われるから危険」
	と判断して止める。
	
	◆ 安全に切り替えられるパターン
	
	未コミット変更があっても…
	
	・　そのファイル自体が切り替え先ブランチには存在しない
	・　同じ行を変更していない
	・　全く別ファイルの変更
	
	こういうときは switch が成功する。
