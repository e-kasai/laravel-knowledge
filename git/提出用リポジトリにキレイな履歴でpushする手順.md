
# 

## 目的
- ごちゃごちゃした履歴を残さず、提出用リポジトリには完成形をpushする
- バックアップの自分用リポジトリには随時pushし万一に備える


## 手順

---
### 0 バックアップ用リポジトリに紐づけ随時pushしながら作業
#### この時点で以下を作っておく

- README.md
- .gitignore
- .gitkeep
---

### 1　コードを完成形にする
- VSCode でコードを完成形にする（git履歴リセットより前に済ませる)
- `.gitignore` を再度確認しておく

---

### 2　git履歴をリセット

- エクスプローラーから該当プロジェクトディレクトリの .gitフォルダを削除する（ゴミ箱に入るので安全）

### 3 git 初期化

```
git init
git branch -M main
```

### 4 提出用リモートリポジトリ紐づけ

```
git remote add origin 提出用リモートリポジトリURL
# push先が提出用リモートリポジトリか確認
git remote -v
```

### 5 add～push
```
#あくまで１例 "add ." ではなくaddするファイルを選ぶ方がベター
git add .
git commit -m "xxx"
```
