## integer = 整数型の比較 （MySQL）


| 型名                     | ストレージサイズ  | 符号付きの範囲                                      | 符号なしの範囲                                                                           |
|--------------------------|------------------|-----------------------------------------------------|----------------------------------------------------|
| `TINYINT`                | 1バイト          | -128 〜 127                                         | 0 〜 255                                           
| `SMALLINT`               | 2バイト          | -32,768 〜 32,767                                   | 0 〜 65,535                                                                    |
| `MEDIUMINT`              | 3バイト          | -8,388,608 〜 8,388,607                             | 0 〜 16,777,215                                                                  |
| `INT`（`INTEGER` 同義）  | 4バイト          | -2,147,483,648 〜 2,147,483,647                     | 0 〜 4,294,967,295                                                      |
| `BIGINT`                 | 8バイト          | -9,223,372,036,854,775,808 〜 9,223,372,036,854,775,807 | 0 〜 18,446,744,073,709,551,615                   


#### 使い分け
- 値が小さい（状態コード・フラグ・性別など）：`TINYINT`
- 数万程度までのカウント：`SMALLINT`
- 数百万〜千万規模：`MEDIUMINT`
- 一般的な ID やカウント（数十億まで）：`INT`
- 非常に大きな累積値や外部連携の大きな識別子：`BIGINT`
- 

#### +や-の符号なしにするcode
```php

//unsignedを使う

$table->unsignedTinyInteger('flag'); //型の前にunsignedを足す
または
$table->tinyInteger('flag')->unsigned(); //メソッドチェーンでも可

```
